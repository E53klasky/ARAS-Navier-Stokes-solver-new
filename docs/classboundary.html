Detailed Description

Contains all the global variables related to the imposing of boundary conditions, and functions to impose BCs.

Depending on the location of the variable in the mesh (face-center, cell-center, etc.), the boundary condition (BC) may either be imposed directly, or through averaging. Moreover, the nature of the boundary condition (Neumann, Dirichlet, etc.) also affects the manner of imposing the BC. First of all, the wall on which the BC must be applied is passed to the constructor as an argument. This information is stored in the wallNum variable, which takes the following values:

  • 0 = left wall along X direction
  • 1 = right wall along X direction
  • 2 = front wall along Y direction
  • 3 = rear wall along Y direction
  • 4 = bottom wall along Z direction
  • 5 = top wall along Z direction

Next the dimension normal to the wall has to be identifed. This is necessary for shifting the view of the wall slice when applying the BC within imposeBC. This value is stored by the shiftDim integer variable, which takes the following values:

  • 0 = X direction
  • 1 = Y direction
  • 2 = Z direction

The view of the wall slice defined by any one of the blitz RectDomain objects in the field::fWalls has to shifted to the left or right in order to access data at the points adjacent to the wall. This is needed when averaging/interpolating the values at the wall. The integer value by which this shifting has to be done is specified by shiftVal, which could be either +1 or -1.

Finally not all subdomains need to have the BC applied to them. The boolean variable rankFlag is set appropriately in the constructor so that imposeBC is calles only for those MPI ranks whose boundaries need the BC applied to them.

Member Function Documentation

void boundary::imposeBC ( )
virtual

Prototype function to impose the boundary conditions on the given field.

Based on the values of wallNum, shiftDim, shiftVal and fieldVal, the appropriate BC (neumann/dirichlet/mixed) is imposed on the field.

Reimplemented in hotPlateCC, neumannFC, neumannCC, periodicFC, periodicCC, dirichletFC, and dirichletCC.
